# 代理人私有 CRM MVP PRD

## 1. 背景

保险代理人的客户资料、画像和跟进记录高度敏感，但这些信息又是代理人日常经营的核心资产。

当前目标是在系统内增加一个简单的客户关系管理子系统，让代理人可以维护自己的客户资料和跟进记录；租户管理员只负责配置本租户可用的客户类型、画像字段和跟进状态，不接触任何代理人的客户数据。

本 PRD 定义 MVP 范围。MVP 优先解决：

- 客户资料统一维护
- 不同客户类型拥有不同画像字段
- 不同客户类型拥有不同跟进状态
- 代理人记录客户跟进过程

MVP 不追求完整销售机会管理、待办提醒、统计看板或端到端加密方案。

## 2. 产品目标

### 2.1 目标

1. 代理人可以创建、查看、编辑、归档自己的客户。
2. 代理人可以按客户类型维护不同画像信息。
3. 代理人可以为客户新增跟进记录，并记录跟进状态和内容。
4. 租户管理员可以预设客户类型、画像字段和跟进状态。
5. 管理员不能查看、搜索、导出任何代理人的客户资料和跟进记录。

### 2.2 非目标

MVP 不包含：

- 端到端加密方案
- 客户共享
- 客户转交
- 管理员查看客户数据
- 管理员统计报表
- 租户级客户去重
- 完整销售机会管理
- 待办和提醒
- 附件、图片、文件上传
- 复杂画像字段类型
- 字段必填、联动、评分、自动标签
- 客户导入导出
- 跟进记录协作、评论、审批

## 3. 用户与权限

### 3.1 代理人

代理人是 CRM 的核心使用者。

代理人可以：

- 创建自己的客户
- 查看自己的客户列表
- 查看自己的客户详情
- 编辑自己的客户基础信息、标签、备注和画像字段
- 归档自己的客户
- 为自己的客户新增跟进记录
- 查看自己的客户跟进时间线

代理人不能：

- 查看其他代理人的客户
- 查看其他代理人的跟进记录
- 配置客户类型
- 配置画像字段
- 配置跟进状态

### 3.2 租户管理员

租户管理员负责配置租户级 CRM 元数据。

管理员可以：

- 创建、编辑、停用客户类型
- 为客户类型配置画像字段
- 为客户类型配置跟进状态
- 配置客户类型是否支持销售机会

管理员不能：

- 查看客户列表
- 查看客户详情
- 查看客户画像
- 查看跟进记录
- 搜索客户
- 导出客户
- 查看客户统计
- 接管或转交客户资料

## 4. 核心概念

### 4.1 客户类型

客户类型由租户管理员统一配置，租户内代理人共同使用。

示例：

- 保险客户
- 招募对象

不同客户类型可以拥有不同画像字段和跟进状态。

客户类型字段：

| 字段 | 说明 |
| --- | --- |
| 类型名称 | 展示给管理员和代理人的客户类型名称 |
| 类型说明 | 管理员理解用，可选 |
| 是否启用 | 停用后不能再新建该类型客户，历史客户保留 |
| 是否支持销售机会 | MVP 仅保存配置，暂不实现完整机会管理 |
| 排序 | 控制代理人端选择顺序 |

### 4.2 画像字段

画像字段是客户类型下的属性化字段。

MVP 中画像字段全部为文本类型，不做结构化数值计算。

字段配置：

| 字段 | 说明 |
| --- | --- |
| 字段名称 | 代理人填写画像时看到的名称 |
| 字段描述 | 解释该字段应该记录什么 |
| 是否启用 | 停用后默认不再展示在编辑表单中 |
| 排序 | 控制展示顺序 |

删除画像字段时不物理删除，只停用或隐藏。历史客户已经填写过的画像值继续保留。

### 4.3 跟进状态

跟进状态按客户类型配置。

示例：

- 保险客户：初聊、需求确认、方案发送、成交、暂缓
- 招募对象：初识、邀约、面谈、待决定、入职

状态配置：

| 字段 | 说明 |
| --- | --- |
| 状态名称 | 新增跟进记录时可选择的状态 |
| 是否启用 | 停用后不能再被新跟进记录选择，历史记录保留 |
| 排序 | 控制展示顺序 |

### 4.4 销售机会

部分客户类型存在多次转化场景，例如保险客户可能先购买医疗险，后续复购其他保险或给家人购买。

部分客户类型不存在销售机会场景，例如招募对象通常不需要多个销售机会。

MVP 只在客户类型上保留“是否支持销售机会”配置，不实现机会列表、机会阶段、机会金额、成交结果，也不要求跟进记录关联机会。

## 5. 客户资料

### 5.1 固定基础字段

客户基础字段与客户类型无关。

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| 姓名/称呼 | 是 | 客户展示名称 |
| 手机号 | 否 | 用于代理人本人名下同类型重复提示 |
| 微信号 | 否 | 用于代理人本人名下同类型重复提示 |
| 客户类型 | 是 | 决定画像字段和跟进状态 |
| 标签 | 否 | 代理人自由标签 |
| 备注 | 否 | 代理人自由文本备注 |
| 最近跟进时间 | 系统维护 | 来自最新一条跟进记录 |
| 归档状态 | 系统维护 | 正常或已归档 |

### 5.2 标签

MVP 中标签由代理人自由输入，不由管理员预设。

标签只属于代理人自己的客户管理语境，不参与租户级统计和配置。

### 5.3 客户类型变更

代理人允许修改客户类型。

规则：

1. 修改客户类型后，详情页和编辑页展示新客户类型的画像字段。
2. 原客户类型下已填写的画像值隐藏保留，不物理删除。
3. 如果后续切回原客户类型，原画像值可以继续展示。
4. 跟进记录历史状态保留原展示文本，不因客户类型变更而丢失语义。

### 5.4 重复提示

MVP 只做代理人本人名下的重复提示。

查重范围：

```text
同一租户 + 同一代理人 + 同一客户类型 + 手机号或微信号
```

规则：

1. 同一个手机号可以同时存在于不同客户类型下。
2. 例如同一个人既可以是“保险客户”，也可以是“招募对象”，这是两个客户。
3. 同一代理人、同一客户类型下录入重复手机号或微信号时，系统提示已存在相同客户。
4. MVP 建议提示但允许继续保存，避免误伤真实业务场景。
5. 不做租户级查重，不提示其他代理人的客户。

## 6. 跟进记录

跟进记录挂在客户下，只能由客户所属代理人创建和查看。

字段：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| 跟进时间 | 是 | 默认当前时间，可修改 |
| 跟进状态 | 是 | 从客户当前类型的启用状态中选择 |
| 跟进内容 | 是 | 文本记录 |

规则：

1. 新增跟进记录后，客户的最近跟进时间更新为最新跟进时间。
2. 跟进状态必须来自该客户类型配置。
3. 如果历史状态后续被管理员停用，历史记录仍展示原状态。
4. MVP 不支持附件、图片、下次跟进时间、提醒或协作。

## 7. 页面需求

### 7.1 小程序端：客户列表

客户列表是代理人的主要入口。

能力：

- 查看自己的正常客户
- 查看已归档客户
- 关键词搜索
- 按客户类型筛选
- 按最近跟进时间排序
- 按新建时间排序

搜索范围：

- 姓名/称呼
- 手机号
- 微信号
- 标签

列表项展示：

- 姓名/称呼
- 客户类型
- 标签
- 最近跟进时间

### 7.2 小程序端：新建客户

流程：

1. 选择客户类型。
2. 填写基础字段。
3. 填写该客户类型启用中的画像字段。
4. 保存时进行本人同类型重复提示。
5. 保存成功后进入客户详情页。

### 7.3 小程序端：客户详情

客户详情页结构：

1. 基础信息区
   - 姓名/称呼
   - 客户类型
   - 手机号
   - 微信号
   - 标签
   - 备注
2. 画像信息区
   - 展示当前客户类型启用中的画像字段
   - 已停用字段默认不展示
3. 跟进记录区
   - 按跟进时间倒序展示
   - 展示跟进时间、状态、内容
4. 操作
   - 编辑客户
   - 新增跟进
   - 归档客户

### 7.4 小程序端：新增跟进

字段：

- 跟进时间
- 跟进状态
- 跟进内容

保存后：

- 新增一条跟进记录
- 更新客户最近跟进时间
- 返回客户详情页

### 7.5 Admin 端：客户类型列表

管理员在 admin 端维护租户级客户类型。

列表展示：

- 类型名称
- 是否启用
- 是否支持销售机会
- 画像字段数量
- 跟进状态数量
- 排序
- 编辑入口

### 7.6 Admin 端：客户类型编辑

编辑内容：

- 基础配置
  - 类型名称
  - 类型说明
  - 是否启用
  - 是否支持销售机会
  - 排序
- 画像字段配置
  - 新增字段
  - 编辑字段名称和描述
  - 调整排序
  - 停用字段
- 跟进状态配置
  - 新增状态
  - 编辑状态名称
  - 调整排序
  - 停用状态

Admin 端不提供客户管理入口。

## 9. 权限与数据隔离

### 9.1 代理人数据隔离

所有客户数据查询必须同时按以下条件约束：

```text
tenantId + agentId
```

代理人不能通过客户 ID、跟进记录 ID 或搜索接口访问其他代理人的数据。

### 9.2 管理员限制

管理员只能访问租户级 CRM 配置数据：

- CustomerType
- CustomerProfileField
- FollowUpStatus

管理员不能访问：

- Customer
- CustomerProfileValue
- FollowUpRecord

### 9.3 敏感数据

MVP 暂不实现端到端加密，但产品上将客户数据视为敏感正式存储数据。

后续设计加密方案时，需要重新评估：

- 哪些字段必须加密
- 哪些字段需要搜索
- 搜索与加密之间的取舍
- 是否保留明文索引或哈希索引
- 代理人换设备、找回账号时的密钥恢复机制

## 10. 核心流程

### 10.1 管理员配置客户类型

1. 管理员进入客户类型配置页。
2. 新建客户类型，例如“保险客户”。
3. 配置画像字段，例如“家庭情况”“保障缺口”“预算偏好”。
4. 配置跟进状态，例如“初聊”“需求确认”“方案发送”“成交”。
5. 保存并启用。
6. 代理人在小程序端可以选择该客户类型创建客户。

### 10.2 代理人创建客户

1. 代理人进入客户列表。
2. 点击新建客户。
3. 选择客户类型。
4. 填写基础信息和画像字段。
5. 系统按本人同类型手机号或微信号检查重复。
6. 如命中重复，展示提示。
7. 代理人确认保存。
8. 系统创建客户并进入详情页。

### 10.3 代理人新增跟进

1. 代理人进入客户详情。
2. 点击新增跟进。
3. 选择跟进状态。
4. 填写跟进时间和内容。
5. 保存。
6. 系统新增跟进记录，并更新客户最近跟进时间。

### 10.4 管理员停用画像字段

1. 管理员进入客户类型编辑页。
2. 停用某个画像字段。
3. 保存。
4. 代理人新建或编辑客户时不再看到该字段。
5. 历史客户已填写的字段值保留，不物理删除。

## 11. 验收标准

### 11.1 配置能力

- 管理员可以创建至少两个客户类型。
- 不同客户类型可以配置不同画像字段。
- 不同客户类型可以配置不同跟进状态。
- 管理员可以停用客户类型、画像字段和跟进状态。
- 停用配置不删除历史数据。

### 11.2 客户管理

- 代理人可以创建客户。
- 代理人可以查看自己的客户列表。
- 代理人可以查看自己的客户详情。
- 代理人可以编辑自己的客户基础信息和画像字段。
- 代理人可以归档自己的客户。
- 代理人不能看到其他代理人的客户。
- 管理员不能看到客户列表和客户详情。

### 11.3 重复提示

- 同一代理人、同一客户类型下，手机号重复时有提示。
- 同一代理人、同一客户类型下，微信号重复时有提示。
- 同一手机号在不同客户类型下允许创建两个客户。
- 系统不提示其他代理人的重复客户。

### 11.4 跟进记录

- 代理人可以为自己的客户新增跟进记录。
- 跟进状态来自客户当前类型配置。
- 客户详情展示跟进记录时间线。
- 新增跟进后客户最近跟进时间更新。
- 管理员不能查看跟进记录。

### 11.5 销售机会预留

- 客户类型可以配置是否支持销售机会。
- MVP 中该配置不影响客户创建和跟进记录创建。
- 关闭销售机会的客户类型不展示任何机会相关入口。

## 12. 后续阶段

### 12.1 销售机会管理

仅对启用销售机会的客户类型开放。

可能能力：

- 一个客户拥有多个销售机会
- 机会名称
- 机会阶段
- 预计金额
- 成交结果
- 跟进记录可关联机会

### 12.2 待办提醒

在跟进记录上增加下次跟进时间，生成代理人自己的待办列表。

### 12.3 数据安全增强

根据真实使用情况设计字段级加密或端到端加密。

重点评估：

- 客户姓名、手机号、微信号、画像、跟进内容的加密策略
- 搜索能力如何保留
- 密钥如何恢复
- 多设备如何同步

### 12.4 导入导出

只允许代理人导入和导出自己的客户数据。

### 12.5 匿名统计

如果后续需要管理员看经营情况，只考虑不可回溯到客户内容的聚合统计，并单独设计隐私边界。
